ER Waiting Room Management

Manage

Objective

Create a web-based application for managing the waiting room of an emergency department, allowing staff to track patients, prioritize care, and manage room assignments efficiently.



Technologies Used

  • HTML
  • CSS
  • JavaScript
  • PHP
  • Postgres

Features

  1. Patient Registration: Form to register new patients with details like name, symptoms, and priority level.
  2. Patient List: Display a list of patients currently in the waiting room, sortable by priority, time of arrival, etc.
  3. Patient Details: View detailed information of each patient.
  4. Room Assignment: Admit patients to available rooms based on priority.
  5. Notifications: Alert staff about high-priority patients or room availability.
  6. Backend Integration: PHP backend to handle patient data, room admission, and notifications.

Architecture

  • HTML: Structure the application layout using HTML elements.
  • CSS: Style the application for a visually appealing and user-friendly interface.
  • JavaScript: Implement interactive features and real-time updates.
  • PHP: Backend logic for managing patient data and room assignments.
  • Postgres: Database to store patient records and room information.

Technical Details

  1. Module 1: Initialization
    • Create the main layout with HTML and CSS.
    • Initialize variables to track patient data, room availability, and notifications.
  2. Module 2: Patient Registration
    • Implement a form for registering new patients.
    • Validate and submit patient data to the backend.
  3. Module 3: Patient List
    • Display a sortable list of patients.
    • Use JavaScript to update the list dynamically.
  4. Module 4: Patient Details
    • Create a detailed view for each patient.
  5. Module 5: Room Assignment
    • Implement room assignment logic.
  6. Module 6: Notifications
    • Use JavaScript to display alerts and updates.
  7. Module 7: Backend Integration
    • PHP scripts to handle patient data and room assignments.
    • Use AJAX for seamless communication between frontend and backend.

Data Flow

  • Staff input patient details, which are sent to the backend and stored in the database.
  • Data is fetched from the database and displayed dynamically on the frontend.
  • Staff admit patients into rooms, updating the backend and reflecting changes in real-time.

Testing

Perform manual testing to ensure the functionality of patient registration, list updates, and notifications. Test backend integration to ensure smooth data handling between frontend and backend.
